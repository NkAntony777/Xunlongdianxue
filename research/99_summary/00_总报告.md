# 寻龙点穴引擎 - 调研总报告

> **项目代号**：Xunlong Engine（寻龙引擎）
> **版本**：v0.1 调研
> **日期**：2026-07-14
> **目标**：在指定区域，基于 DEM 与水系等数据，按"四兽"与"龙砂穴水向"等风水参数，自动分析并输出候选龙穴地点 + 综合评分。

---

## 0. 摘要

通过对传统风水（形势派 / 峦头派）核心理论与 DEM / GIS 算法的双向调研，本报告得出以下结论：

1. **传统术语可以 100% 算法化**：龙砂穴水向、五星、剥换、过峡、入首、四兽、明堂、水口、玉带水等概念，都能用现代 GIS 概念一对一映射（详见 `01_core_theory/`、`04_acupoint/`）。
2. **已有高相关开源实现**：limi124/shanshui-mingtang-fengshui-gis（51 stars）的算法可直接参考；MDPI / Nature 上有 3+ 篇学术论文。
3. **算法基础已成熟**：山脊线提取（D8 反向 / 特征显著度法）、TPI、TWI、Concave Hull、Viewshed、KDE 均有成熟 Python 实现。
4. **存在量化空白区**："理气派"（三元九运、玄空飞星）需要时间维度数据，**v1.0 不实现**，留为 v2.0 扩展。

## 2. 传统理论 → GIS 算法映射总表

| 传统概念 | GIS / 数学概念 | 算法 / 库 | 优先级 |
|---------|---------------|---------|-------|
| **龙脉** | 山脊线（Ridge） | `pysheds` (flow_acc=0) / `richdem` | P0 |
| **太祖山** | 一级分水岭起点 | 山脊线 + Strahler 分级 | P1 |
| **父母山** | 末端山脊线最大高程点 | 山脊线末端追踪 | P1 |
| **入首** | 山脊线最末分支起点 | 山脊线 + 曲率检测 | P1 |
| **过峡** | 山脊线鞍部点 | 山脊线极小值检测 | P2 |
| **剥换** | 山脊线曲率突变 | 沿脊线窗口方差 | P2 |
| **青龙 / 白虎** | 候选穴东/西扇区局部高程 | 扇区统计 | P0 |
| **朱雀 / 玄武** | 候选穴南/北扇区坡度+视野 | 扇区 + Viewshed | P0 |
| **砂山** | 候选穴周围 360° 高点 | 邻域峰值检测 | P0 |
| **明堂** | 候选穴前开阔区 | 扇区平均坡度 | P0 |
| **案山 / 朝山** | 候选穴前显著山 | 距离分层的局部极大值 | P1 |
| **靠山** | 候选穴后最高峰 | 北向扇区高程 | P0 |
| **水口** | 流域出水口 + 两侧山 | `pysheds` catchment + 邻域山 | P1 |
| **玉带水** | 候选穴前河曲凹侧 | shapely + 曲率检测 | P1 |
| **反弓水** | 候选穴前河曲凸侧 | 同上 | P1 |
| **TPI** | 局部高程相对值 | `scipy.ndimage` | P0 |
| **TWI** | 地形湿度指数 | `pysheds` | P0 |
| **Concave Hull** | 凹包 | `shapely.concave_hull` | P1 |
| **唇毡** | 穴前缓坡 | 扇区坡度方差 | P2 |
| **理气派**（v2.0） | 时间维度三元九运 | 需新增时间数据 | P3 |

## 4. 关键算法选型（已确认）

| 任务 | 选定算法 | 原因 |
|------|---------|------|
| 流向分析 | D8 (`pysheds`) | 稳定、Python 原生 |
| 山脊线提取 | 汇流累积=0 + 骨架化 | 最简单、抗噪 |
| 形态识别 | TPI + Concave Hull | 经典、稳定 |
| 水系评分 | 距离 + 方向（参考 shanshui-mingtang-fengshui-gis） | 已有工程实现 |
| 视域分析 | 简化版（多方向射线） | 避免 GPU 依赖 |
| 综合评分 | 加权融合（参考 shanshui-mingtang-fengshui-gis） | 透明、可调 |

## 5. 评分公式（推荐）

```python
overall_score = (
    0.20 * back_mountain_score      # 玄武（靠山）
  + 0.20 * qinglong_baihu_score    # 龙虎（左右）
  + 0.18 * zhuque_score            # 朱雀（明堂）
  + 0.15 * form_score              # 穴位形态
  + 0.12 * water_score             # 得水
  + 0.15 * stability_score         # 稳定性
)
```

各子分项范围 0-100，权重和为 1.0。

## 7. 关键技术风险

| 风险 | 缓解 |
|------|------|
| DEM 精度不够导致误判 | 提示用户上传 ≥ 12.5 m 分辨率 DEM |
| 水系数据缺失 | 允许用 DEM 派生水系（汇流累积）作为替代 |
| 算法权重不合理 | 暴露权重参数 + 允许用户自定义 |
| LLM 报告可信度 | 严格控制 prompt，禁止凭空预测 |
| 性能（GB 级 DEM） | 切片 + 分块 + 异步处理 |
| 跨域（CRS） | 一律转为 EPSG:3857 进行距离计算 |
