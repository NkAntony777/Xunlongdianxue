# 03 - 龙脉识别的地形学 / GIS 方法

> 把"寻龙"翻译为"山脊线提取 → 分级 → 蜿蜒度评估 → 入首点定位"四步流程。

## 1. 概念对应

| 传统术语 | 地形学 / GIS 对应 |
|---------|-----------------|
| 龙脉 | 山脊线 / 分水岭（ridge / divide） |
| 主脉 | 一级分水岭（最高级山脊） |
| 支脉 | 二级及以下分水岭 |
| 剥换 | 山脊线上的曲率突变、坡度阶跃 |
| 过峡 | 山脊线上的鞍部点（saddle） |
| 入首 | 山脊线末端 → 即将终止的位置 |
| 结穴 | 末端低洼地形 + 周围山势环抱 |

## 2. 山脊线提取的五大算法（综述）

### 2.1 坡向变率法（Profile / Plan Curvature）

- 原理：山脊线处**剖面曲率为极大正值**，山谷线处为极小负值。
- 公式：曲率 `k = -d²z/dx² / (1 + (dz/dx)²)^1.5`
- 优点：几何意义清晰；缺点：对 DEM 噪声敏感。

### 2.2 坡向法（Aspect-based）

- 原理：山脊线两侧坡向相对（180° ± ε），山谷线两侧坡向相向。
- 优点：对噪声鲁棒；缺点：平坦地区不可用。

### 2.3 水文分析法（Flow-routing / D8 反向）

- 原理：山脊线 = **汇流累积量为 0** 的栅格（因为水流总是从山脊两侧分流）。
- 实现：D8 流向 + 汇流累积 → 累积=0 提取 → 反地形同样可提取山谷线。
- **本引擎推荐**（最稳定、最常用）。

### 2.4 断面高程极值法（Profile Maxima）

- 原理：沿东西 / 南北 / 任意方向切断面，提取断面上的极大值点连成山脊线。
- 缺点：方向选择有偏，会丢失细节。

### 2.5 尺度空间 + 多角度（Scale-space & Multi-angle）

- 林铁等 2016 提出：先做高斯金字塔降采样 + 粗到细提取 + 多角度断面增强。
- 优点：抗噪声，特征完整性好。
- 缺点：实现较复杂。

### 2.6 特征显著度法（Feature Significance）

- 胡光慧等：先滤波 + 降采样，再计算每个候选特征点的"显著度"（高度 + 落差 + 坡度综合），按显著度阈值裁剪。
- 优点：可控性强、可由用户调节"想要多少条山脊"。

## 3. 算法选型（推荐）

| 场景 | 推荐算法 | 库 / 工具 |

| 快速原型 | 水文分析（D8 反向） | `pysheds`, `richdem`, `whitebox` |

## 4. 寻龙四步流程（推荐方案）

```
Step 1. 数据预处理
    ↓
    DEM（GeoTIFF） → Pit Fill（填洼）→ D8 流向 → 汇流累积
    ↓
Step 2. 山脊线提取
    ↓
    汇流累积 == 0 提取 → 矢量化 → 简化（Douglas-Peucker）
    ↓
Step 3. 龙脉分级
    ↓
    计算每条山脊线的：长度、蜿蜒度、平均高程、最大落差
    → 按"流路从源头到末端"建立树状结构
    → 太祖山（根）→ 少祖山 → 父母山（叶）
    ↓
Step 4. 入首点定位
    ↓
    末端山脊线节点 + 局部曲率突变 → 入首点
    → 入首点周围 100-500m 搜索候选穴
```

## 5. 关键算法细节

### 5.1 Pit Fill（填洼）

- 必须先填洼，否则 D8 流向会出现死循环。
- 实现：`pysheds.grid.fill_depressions` 或 `whitebox.FillDepressions`。

### 5.2 D8 流向编码

```
N  NE  E  SE  S  SW  W  NW
64 128  1  2  4  8  16  32
```

- 每个栅格的水流方向编码为 8 个方向之一（最大坡降方向）。

### 5.3 汇流累积

- 累积 = 上游有多少栅格的水流汇入本栅格。
- **山脊线定义**：累积 = 0 的栅格（无上游水流，仅起分水作用）。

### 5.4 山脊线矢量化

- 提取累积=0 的栅格 → 转成 polyline（使用栅格邻接追踪）。
- 推荐用 `skimage.measure.find_contours` 或自实现连通域追踪。

### 5.5 蜿蜒度（Smoothness / Sinuosity）

```
蜿蜒度 = 山脊线总长 / 起点到终点直线距离
```

- 真龙：蜿蜒度 ≥ 1.3（参考阈值）
- 假龙 / 直硬龙：蜿蜒度 < 1.1

### 5.6 剥换检测

- 沿山脊线移动窗口（窗口大小 = 1/10 山脊线总长），计算高程方差。
- 方差突变点 = 剥换位置。
- 量化：剥换次数 ≥ 3 为贵。

## 7. Python 伪代码示例

```python
import numpy as np
import rasterio
from pysheds.grid import Grid
from skimage.morphology import skeletonize

# 1. 加载并填洼
grid = Grid.from_raster("dem.tif", data_name="dem")
grid.fill_depressions("dem", out_name="flooded")
grid.resolve_flats("flooded", out_name="inflated")

# 2. D8 流向
dirmap = (64, 128, 1, 2, 4, 8, 16, 32)
grid.flowdir("inflated", out_name="dir", dirmap=dirmap)

# 3. 汇流累积
grid.accumulation("dir", out_name="acc", dirmap=dirmap)

# 4. 山脊线栅格（汇流累积 == 0）
ridge_mask = (grid.acc == 0) & np.isfinite(grid.dem)
ridge_mask = skeletonize(ridge_mask)  # 细化到单像素宽
```

## 8. 山脊线矢量化（细化 + 追踪）

```python
from skimage.measure import label, regionprops
from shapely.geometry import LineString, Point
import numpy as np

def raster_ridge_to_lines(ridge_mask, transform, min_length=20):
    """将山脊线栅格转为 LineString 列表。"""
    labeled = label(ridge_mask, connectivity=2)
    lines = []
    for region in regionprops(labeled):
        if region.area < min_length:
            continue
        # 提取骨架坐标
        coords = region.coords  # (row, col)
        xy = [rasterio.transform.xy(transform, r, c) for r, c in coords]
        if len(xy) < 2:
            continue
        lines.append(LineString(xy))
    return lines
```

## 10. 实际工程要点

1. **DEM 分辨率**：建议 5-30 m。高分辨率（<5 m）会出现伪山脊线，需要先平滑。
2. **平滑预处理**：3×3 或 5×5 高斯滤波。
3. **山脊线抽稀**：Douglas-Peucker，容差 = 1-2 个像元。
4. **山脊线分级**：可用"上游累积栅格数"作为分级的代理变量（类似 Strahler 河流分级）。
5. **入首点判定**：在山脊线最末端 1/10 长度内，曲率极大值 + 高程急降点 = 入首候选。
