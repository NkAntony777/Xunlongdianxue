## 2. 高相关开源项目

### 2.1 limi124/shanshui-mingtang-fengshui-gis（GitHub, 51 stars）

- **定位**：AI 辅助的 WebGIS 风水评估平台
- **技术栈**：
  - 前端：Vue 3 + Vite + OpenLayers + CesiumJS + ECharts
  - 后端：Python + FastAPI + GeoPandas + Rasterio + Shapely + NumPy
- **核心模块**（来自 services/）：
  - `terrain_analysis.py`：DEM 高程、坡度、坡向、地形起伏、粗糙度
  - `water_analysis.py`：水系距离 / 方向 / 评分
  - `fengshui_score.py`：靠山 / 明堂 / 藏风 / 得水 / 采光 / 稳定性 六维评分
  - `report_generator.py` + `llm_report_generator.py`：传统报告 + LLM 解读
- **核心公式**（来自 `fengshui_score.py`）：

```python
back_mountain_score = clamp_score(68 + min(relief / 4, 20) - max(slope - 15, 0))
front_open_score = clamp_score(82 - slope * 1.1 - roughness * 12)
enclosure_score = clamp_score(70 + min(relief / 12, 12) - max(roughness - 0.5, 0) * 30)
aspect_light_score = 86 if aspect in {"南", "东南", "西南"} else 72 if aspect in {"东", "西"} else 58
terrain_stability_score = clamp_score(92 - slope * 1.8 - max(max_slope - 25, 0) * 1.2)

weights = {
    "back_mountain_score": 0.2,
    "front_open_score": 0.18,
    "enclosure_score": 0.17,
    "water_score": 0.12,
    "aspect_light_score": 0.15,
    "terrain_stability_score": 0.18,
}
```

- **可借鉴度**：★★★★★（直接参考核心算法 + UI 设计）

## 5. 关键技术难点与解决方案

| 难点 | 解决方案 |
|------|---------|
| DEM 数据量大（GB 级） | 切片 + 多分辨率金字塔 + 分块处理 |
| 流向分析死循环（平地） | `resolve_flats` 算法 |
| 山脊线过密 | 特征显著度阈值裁剪 + Douglas-Peucker 抽稀 |
| 视域分析性能 | 多线程 / GPU（CUDA） / 简化地形 |
| 四兽方向依赖坐向 | 允许用户定义坐向，或自动从主脉方向推断 |
| 多源 DEM 拼接 | `gdalbuildvrt` + 重投影 |
| 候选穴排序 | 加权综合评分 + 非极大值抑制 |
| 三维可视化 | Cesium + TerrainProvider + LOD |
