# 得水与水煞：分离模型（传统要点 → 可计算）

> 调研日期：2026-07-14（AnySearch + 经典水法整理）  
> **核心主张**：得水加分与水煞扣分是**两条独立维度**，不可用「距水远近」单一函数混算。  
> 性质：学术/工程对照，非决策建议。

---

## 1. 为何必须分开

《葬经》曰：**「风水之法，得水为上，藏风次之。」**

- **得水**：证明生气旺盛、气有所止（界水）、财禄之象（「山管人丁水管财」的水侧）。  
- **水煞**：水（或路之「虚水」）形态/距离/方位无情，冲散或割裂穴气。

民间常混谈「近水好」「近水凶」。精确说法是：

```text
有水且有情  → 得水（吉）
有水但无情  → 水煞（凶）
无水         → 不得水（中性偏弱，非必然大凶）
贴水/在水中 → 常兼割脚 + 不得「陆上结穴」
```

**同一段河**，对凹岸（环抱侧）是玉带得水，对凸岸（弧背侧）是反弓水煞——**位置不同，吉凶相反**。

## 2. 得水（吉水）体系

### 2.1 经典吉格

| 名称 | 形态特征 | 传统评价 | 几何可检 |
|------|----------|----------|----------|
| **玉带水 / 金城水** | 弧线环抱穴，弓内侧向穴 | 大吉、旺财 | 局部曲率，曲率中心与穴同侧 |
| **聚堂水 / 水聚天心** | 明堂中正处停蓄汇聚 | 巨富贵，优于单朝 | 前扇区水体面积大、流速代理低 |
| **九曲 / 之玄** | 多弯缓行 | 聚气、贵秀 | 折点数、曲折度 |
| **朝水** | 特来当面，宜有案砂不冲割 | 催官催富；急射则转凶 | 来水方向对向、夹角、是否被案隔 |
| **横水局** | 水城横绕 | 可吉 | 横切明堂、不破心 |
| **暗拱** | 穴上不见水，砂外聚/朝 | 「明朝不如暗拱」 | 近案遮挡 + 外侧水体 |
| **水缠玄武** | 水绕穴后 | 发福久 | 后向有水环 |
| **天门开、地户闭** | 来远去关 | 源远流长、财不漏 | 来水可视开阔 + 去水关锁 |

口诀方向：

- 「朝水不如聚水」  
- 「其去欲流，顾我欲留」  
- 来水宜不见源（天门开），去水宜重锁（地户闭）

### 2.2 得水的距离逻辑（非「越近越好」）

| 距离感（量级） | 传统 | 算法建议 |
|----------------|------|----------|
| 水中 / 漫滩 | 非陆上结穴 | **硬禁**（mask） |
| 贴基、影映水中 | 近乎割脚 | 不得水或得水≈0，见水煞 |
| **有余气毡唇后的界水** | 有情 | 得水高区（常见约百米～千米级，随 AOI 缩放） |
| 过远不见水 | 气弱、失界 | 得水衰减 |

> 开源 shanshui 的 50–1000 m 高分带，对应的是 **「近而不贴」的界水**，应归入**得水**，不是水煞带本身。

### 2.3 得水评分通道（建议 API）

```text
get_water_score(dist, form, outlet, mingtang) ∈ [0, 100]
```

输入建议：

1. `dist`：到有情水体的距离（可只计「面朝扇区」内水体）  
2. `form`：抱/聚/曲/横/朝 的形态分  
3. `outlet`：地户关锁分  
4. `mingtang`：水是否到堂、是否破天心  

**只奖励吉形态**；无情水 `form` 低时得水不应虚高。

## 3. 水煞（凶水）体系

### 3.1 常见「恶水 / 水煞」（形峦）

综合通行水法叙述（七恶水等说法在民间文本中互有出入，下表取共识）：

| 名称 | 特征 | 传统忌 | 几何可检 |
|------|------|--------|----------|
| **反弓水** | 弧背对穴（与玉带相反） | 无情、破财、离心 | 曲率中心与穴**异侧** |
| **直冲 / 冲心 / 水破天心** | 水直射穴或明堂中心 | 散气败财 | 线段指向穴，夹角小 |
| **射胁** | 斜射穴侧 | 凶 | 指向龙虎侧 |
| **割脚水** | 过近贴基、无余气 | 损丁、根基不稳 | dist 极小 + 无毡唇高地 |
| **斜飞 / 直流** | 斜去或一泄无收 | 散、败 | 去水方向、曲折度低 |
| **反跳** | 到堂又折走 | 不到堂 | 折线在堂前外折 |
| **漏槽 / 元辰直泻** | 龙虎内水直出无关 | 气不聚 | 内堂出水无砂截 |
| **扫帚水** | 去水口散乱多路 | 财不聚 | 多出口、关锁弱 |
| **牵牛水** | 顺坡泻去 | 难聚 | 前低且去水顺势 |
| **淋头水** | 后高水压顶 | 压、扰 | 后向高位水体/跌水 |

**玉带 vs 反弓**：弧度相反；同一弯道两侧吉凶对倒。  
**割脚 vs 反弓**：割脚论**距离过近**；反弓论**曲率朝向**；可同时犯（近 + 反弓）。

### 3.2 水煞评分通道（建议 API）

```text
sha_water_penalty(dist, form_bad, rush, reverse_bow) ∈ [0, 100]
# 越高越凶
```

与得水**相减或分权**，例如：

```text
water_term = w_g * get_water_score - w_s * sha_water_penalty
# 或
total *= (1 - α * sha/100)   # 仅煞衰减
total += β * get_water        # 仅得水加成
```

**禁止**：`score = f(dist)` 同时表达二者。

### 3.3 「水煞影响带」可视化 vs 评分

| 概念 | 含义 | 建议 |
|------|------|------|
| 紫灰点阵（参考图） | 河岸邻近**风险/影响示意带** | 图层名可称「近水影响带」；内缘≈割脚风险，外缘≠全面凶 |
| 评分水煞 | 仅对**无情形态 + 过近**重罚 | 外缘轻罚或不罚，避免「整条河谷无穴」 |

## 4. 与《葬经》「得水为上」的关系

「得水为上」强调的是：

1. 水是生气外显、气止之界——**有水局优于无水局**（在龙真穴的前提下）；  
2. **不是**「贴水、冲水、反弓也叫得水」。

故工程表述应为：

```text
优先：有「有情得水」的陆上穴
同时：水煞独立惩罚，贴岸/反弓/直冲不得因「有水」加分
```

## 5. 本引擎对照（实现状态 → 目标）

| 项目 | 当前倾向 | 目标 |
|------|----------|------|
| 水面 + buffer | 硬禁 nan | 保留 |
| 距水函数 | 得水与煞合成在一个 `water_score_from_dist` | **拆成 get + sha 两通道**再合成 |
| 玉带/反弓 | 文档有、栅格场弱 | 河段曲率 + 穴相对位置 |
| 直冲射胁 | 未系统做 | 线段方向 vs 穴 |
| 水口关锁 | 未系统做 | 去水侧 DEM 峰值/砂 |
| 紫灰图层 | 全 buffer 点阵 | 可与 sha 热度分层（内深外浅） |

## 6. 伪代码（推荐）

```python
def water_terms(hole, rivers, facing):
    dist = dist_to_water(hole, rivers)
    if on_water_or_floodplain(hole): 
        return dict(get=0, sha=100, hard_ban=True)

    form = classify_water_form(hole, rivers, facing)
    # form: jade_belt | pool | meander | reverse_bow | rush | cut_foot | plain ...

    get = 0.0
    if form in GI_FORMS:
        get = distance_get_curve(dist) * form_quality(form)
        get *= mingtang_water_bonus(hole, rivers)

    sha = 0.0
    if form in SHA_FORMS:
        sha = form_sha_weight(form)
    sha = max(sha, cut_foot_sha(dist))  # 过近强制抬煞
    sha = max(sha, rush_sha(hole, rivers))

    return dict(get=get, sha=sha, hard_ban=False)

# 综合
water_contrib = w_get * get - w_sha * sha
```

## 7. 验收标准（产品）

1. 凹岸高台：得水↑、水煞↓ → 可出峰值  
2. 凸岸/反弓侧：得水↓、水煞↑ → 不当峰值  
3. 河心/漫滩：hard_ban  
4. 贴岸无余气：sha 高，即使得水基线不为 0  
5. 图例：紫灰带 ≠「得水热力」；橙心热力应反映合成场
