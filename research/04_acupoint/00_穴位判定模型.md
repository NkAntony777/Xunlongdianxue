# 04 - 穴位判定规则与数理模型

> 点穴 = 在入首点附近搜索符合"藏风聚气"约束的局部地形。本章给出可量化的判定规则。

## 1. 穴的核心要求（传统口诀 + 算法化）

| 传统要求 | 现代算法化描述 |
|---------|---------------|
| **三合**（龙、穴、砂） | 候选穴必须在主脉末端的山势环抱中（4 方向都有山） |
| **藏风** | 候选穴周围 360° 局部凹凸指数（concave index）≥ 阈值 |
| **聚气** | 候选穴位于汇流累积值较大的低洼区 |
| **界水** | 候选穴前方有水系环抱（河流、沟渠） |
| **乘生气** | 候选穴周围土质好、土壤异常（五色土）— v2.0 卫星影像分析 |
| **前朝** | 候选穴南向视野开阔 + 有朝山 / 案山 |
| **后靠** | 候选穴北侧有父母山（高程差 30-200 m） |
| **左右抱** | 候选穴东西侧砂山有情（青龙高、白虎低） |

## 2. 穴位几何形态判定（"窝钳乳突"四形）

### 2.1 窝穴（Open Concave）

- **几何特征**：中心低洼、四周高起、剖面呈 U 形
- **算法判定**：
  - 候选点周围 30-100 m 内高程方差 > 5 m
  - 候选点高程 = 邻域极小值
  - 邻域内**最低高程位于中心**、最高高程在边缘
  - **Concave hull ratio**（凹包面积 / 凸包面积） < 0.85

### 2.2 钳穴（V-shape Notch）

- **几何特征**：两侧高中间低、U 形剖面 + 突出"钳口"
- **算法判定**：
  - 候选点位于两侧山脊线 U 形交汇处
  - 沿东-西方向剖面呈明显 V 形
  - 钳口方向（开口方向）一般朝南或东南

### 2.3 乳穴（Convex / Promontory）

- **几何特征**：山体延伸出一个乳状突起
- **算法判定**：
  - 候选点位于山脊线末端的凸出部位
  - 局部高程极大值 + 周围有低洼环绕
  - 三面环抱（缺一面）

### 2.4 突穴（Knoll / Isolated Hill）

- **几何特征**：孤立小丘、周围平坦或低洼
- **算法判定**：
  - 候选点局部高程极大值
  - 周围 50-200 m 内高程方差 < 5 m（较平）
  - **Topographic Position Index (TPI)** > 1.5

## 3. 核心算法指标

### 3.1 Topographic Position Index (TPI)

```
TPI = z_0 - mean(z_in_window)
```

- TPI > 0：候选点高于周围（凸形、岗地）
- TPI < 0：候选点低于周围（凹形、盆地）
- 0 附近：平坦地形

**应用**：TPI < -0.5 且周围有屏障 → 窝穴；TPI > 1.0 → 突穴 / 乳穴。

### 3.2 Topographic Wetness Index (TWI)

```
TWI = ln(a / tan(β))
```

- a：上游汇流累积面积
- β：坡度
- TWI 越大 → 越容易积水 → 越"聚气"
- **应用**：TWI 高分位（>8）= 强聚气

### 3.3 Concave Hull / Concavity Index

- 给定候选点周围 N 个高程点，计算 concave hull 与 convex hull 的面积比。
- 比值 < 0.7 → 强凹形（聚气）
- 比值 > 0.95 → 凸形或平坦

```python
from shapely import MultiPoint, concave_hull, convex_hull

def concavity_index(points, ratio=0.3):
    """points: List of (x, y, z) tuples
       ratio: concave hull 参数，0-1"""
    mp = MultiPoint([(p[0], p[1]) for p in points])
    ch = concave_hull(mp, ratio=ratio)
    cvh = convex_hull(mp)
    return ch.area / cvh.area
```

### 3.5 曲率（Curvature）

- 候选穴周围剖面曲率 / 平面曲率
- 窝穴：剖面曲率 < 0（凹陷）
- 突穴 / 乳穴：剖面曲率 > 0（凸起）

### 3.6 相对高差（Relief Ratio）

```
Relief Ratio = (max_elev - candidate_elev) / 距离
```

- 候选穴靠山方向 50-500 m 内高差 / 距离应在 0.1-0.5 范围
- 过陡 = 逼山凶；过缓 = 脱气凶

## 4. 候选穴综合评分公式（推荐）

```python
def candidate_score(candidate, dem, rivers, bbox):
    """候选穴综合评分（0-100）。"""
    # 1. 四象评分（来自 chapter 02）
    four_beasts = score_four_beasts(candidate, dem)  # 0-100

    # 2. 形态评分（窝钳乳突）
    tpi = compute_tpi(dem, candidate, radius=100)
    form_score = classify_form(tpi)  # 0-100

    # 3. 聚气评分（汇流累积 + TWI）
    twi = compute_twi(dem, candidate)
    catchment = flow_accumulation_at(candidate)
    gather_score = normalize(twi * np.log1p(catchment), max=10)

    # 4. 得水评分（最近水系距离 / 方向）
    water_score = score_water(candidate, rivers)

    # 5. 视野开阔度
    viewshed = compute_viewshed(dem, candidate, max_dist=2000)
    openness = score_openness(viewshed)  # 朱雀方向开阔

    # 6. 稳定性（坡度）
    slope = compute_slope(dem, candidate, radius=50)
    stability = 100 - normalize(slope, max_deg=25)

    # 加权
    weights = {
        'four_beasts': 0.30,
        'form': 0.15,
        'gather': 0.15,
        'water': 0.15,
        'openness': 0.10,
        'stability': 0.15,
    }
    overall = sum([
        weights['four_beasts'] * four_beasts,
        weights['form'] * form_score,
        weights['gather'] * gather_score,
        weights['water'] * water_score,
        weights['openness'] * openness,
        weights['stability'] * stability,
    ])
    return overall, {
        'four_beasts': four_beasts,
        'form': form_score,
        'gather': gather_score,
        'water': water_score,
        'openness': openness,
        'stability': stability,
    }
```

## 5. 候选穴排名（Top-K 提取）

```python
def find_top_candidates(dem, rivers, top_k=10, min_score=70, min_distance=200):
    """在整个 DEM 上滑动窗口搜索候选穴。"""
    h, w = dem.shape
    candidates = []
    for r in range(50, h-50, 20):  # 步长 20 m
        for c in range(50, w-50, 20):
            cand = (c, r)
            score, breakdown = candidate_score(cand, dem, rivers, None)
            if score >= min_score:
                candidates.append((cand, score, breakdown))

    # 非极大值抑制：过滤距离过近的候选
    candidates = non_max_suppression(candidates, min_distance=min_distance)

    # 排序
    candidates.sort(key=lambda x: -x[1])
    return candidates[:top_k]
```

## 6. 穴位朝向（坐向）判定

### 6.1 坐向 = 候选穴到案山 / 朝山的方向

- 算法：从候选穴出发，向 135°-225°（南向）扇区寻找最近显著高点 = 朝山方向
- 坐山方向 = 朝山方向反向 ± 180°

### 6.2 罗盘 24 山向

- 24 山向 = 360°/24 = 15° 一个卦位
- 实际精度：2.5°（每山 6 爻）

```python
def degree_to_24_mountain(degree):
    """把 0-360° 角度转换为 24 山向名称。"""
    mountains = ['壬', '子', '癸', '丑', '艮', '寅', '甲', '卯', '乙', '辰',
                 '巽', '巳', '丙', '午', '丁', '未', '坤', '申', '庚', '酉',
                 '辛', '戌', '乾', '亥']
    idx = int(((degree + 7.5) % 360) / 15)
    return mountains[idx]
```

## 7. 验证穴位的"十道证"

传统验证有 10 种"证据"：

| 验证项 | 传统称谓 | 现代算法化 |
|-------|---------|----------|
| 1 | 唇毡（穴前平缓区域） | 候选穴前方 30 m 内坡度方差 < 2° |
| 2 | 蝉翼砂（穴左右细薄低山） | 候选穴左右 10-30 m 范围有薄山 |
| 3 | 牛角砂（穴后两山如牛角） | 候选穴正后方有 V 形开口 |
| 4 | 虾须水（穴两侧微流水道） | 候选穴左右有细小汇流路径 |
| 5 | 蟹眼水（穴侧水洼） | 候选穴侧 50-100 m 有局部汇流极值 |
| 6 | 天门 / 地户 | 入口开阔、出口封闭 |
| 7 | 案山 | 候选穴前 200-1000 m 显著山 |
| 8 | 朝山 | 候选穴前 1-5 km 显著山 |
| 9 | 明堂 | 候选穴前开阔平缓区 |
| 10 | 乐山 / 鬼星 | 候选穴后的主山 |
