# 02 - 四兽（青龙 / 白虎 / 朱雀 / 玄武）的方位与量化方法

> 重点：把"左青龙右白虎，前朱雀后玄武"翻译成可计算的"四向局部高程 / 视野 / 距离指标"。

## 1. 四兽方位基线（坐北朝南视角）

| 四兽 | 方位 | 象征 | 现代 GIS 对应 |
|------|------|------|-------------|
| **青龙** | 左（东） | 男性、事业、阳刚 | 候选点东侧扇区（45°-135°，以 0°为北） |
| **白虎** | 右（西） | 女性、桃花、阴柔 | 候选点西侧扇区（225°-315°） |
| **朱雀** | 前（南） | 明堂、口舌、远景 | 候选点南侧扇区（135°-225°） |
| **玄武** | 后（北） | 靠山、稳定、贵人 | 候选点北侧扇区（315°-45°） |

> 注意：在阴宅/阳宅中，"左"指**人面朝前方时**的左手边（坐山朝向的左手），不是绝对东向。算法应允许用户自定义坐向作为参数。

## 2. 高度量化规则（核心口诀）

### 2.1 "宁可青龙高万丈，不可白虎抬头望"

**结论（量化）**：

| 关系 | 量化阈值（建议默认值） |
|------|----------------------|
| 青龙高度 vs 白虎高度 | 青龙可高于白虎 **不限**；白虎严禁高于青龙 |
| 白虎/青龙高度比 | **≤ 0.85** 为吉；**> 1.0** 为"白虎抬头"凶 |
| 青龙/玄武高度比 | 青龙不可高过玄武（主山）：**≤ 0.95** |
| 两侧高度差 | 任意方向上，**最大高差 ≤ 0.5 × 候选穴高程差** |

**3 寸原则**（民间俗例）：东厢比西厢高 3 寸（≈ 10 cm），对应到地形即"两侧高差占总地形的 1-3%"。

### 2.2 "青龙要动，白虎宜静"

- 青龙侧（东）：有蜿蜒起伏、活脉为佳 → 用**山脊线分形维数**量化（>1.3 吉）
- 白虎侧（西）：圆润、俯伏为佳 → 用**山脊线平滑度**量化（分形维数 <1.15 吉）

### 2.3 "青龙蜿蜒，白虎驯俯"

- **青龙吉形**：蜿蜒起伏、有力（局部起伏比 >1.2）
- **白虎吉形**：低伏圆顺（局部起伏比 <1.0）
- **白虎凶形**：尖射、反背、昂头（坡度 >25° 的山尖）

## 3. 距离量化（参考图标准）

参考图显示：左龙右虎通常在候选穴的 **0.5–3 个穴距离** 范围内，距离过近为"逼"，过远为"散"。

| 距离关系 | 评分 |
|---------|------|
| 青龙距离 0.3-1.5 × 候选穴最大高程距离 | 优 |
| 白虎距离 0.5-2.0 × 候选穴最大高程距离 | 优 |
| 一侧砂山缺失 | 凶（"缺角"） |
| 两侧砂山对称 | 中（"宾主不分"） |

## 4. 朱雀（前方明堂）量化

### 4.1 "朱雀翔舞"

- 前方 30°-90° 扇区内，平均坡度应**小**（开阔）。
- 前方 1-3 km 范围内有弯曲河流或低洼明堂。
- **朱雀不可欺主**：前方山体高度 / 候选穴靠山高度 ≤ 0.6。

### 4.2 明堂分级

| 级别 | 范围 | 量化要求 |
|------|------|---------|
| **内明堂** | 穴前 0-200 m | 平整、紧凑、平均坡度 < 8° |
| **中明堂** | 穴前 200-1000 m | 开阔、缓坡（< 12°） |
| **外明堂** | 穴前 1-5 km | 有案山 / 朝山环抱，朝向秀丽 |

### 4.3 朝山 / 案山

- **案山**：穴前近山，距 200-1000 m，高度不超过 0.5 × 靠山。
- **朝山**：穴前远山，距 1-5 km，形态端正有情。
- 算法：用 **Viewshed** 在候选穴位置朝 135°-225° 方向计算可视范围，识别朝案。

## 5. 玄武（后方靠山）量化

### 5.1 "玄武垂头"

- 靠山方向（背向候选穴）的平均坡度应**缓降**（5°-25°），忌悬崖（>35°）或反弓。
- 靠山高度 ≥ 候选穴高程 + 30-100 m 范围为佳。
- **主山距候选穴**：建议 50-500 m 范围内；过近为"逼山"，过远为"脱气"。

### 5.2 父母山验证

- 算法：在候选穴正北方向 30° 扇区内，找到最大高程点 P_max。
- 验证 P_max 与候选穴距离 d 满足：50 ≤ d ≤ 500 m，且 P_max 高于候选穴 30-200 m。

## 6. 四象评分函数（参考实现，伪代码）

```python
def score_four_beasts(candidate, dem, aspect_radius=300):
    """四象综合评分（0-100）"""
    score = {}

    # 0. 决定坐向：候选穴周围最大坡度反方向
    facing = dominant_aspect(dem, candidate)  # 朝向

    # 1. 青龙（左侧 90° 扇区）
    qinglong_zone = sector(candidate, facing - 90, facing, radius=aspect_radius)
    ql_height = max_height(qinglong_zone)
    ql_roughness = std_height(qinglong_zone)

    # 2. 白虎（右侧 90° 扇区）
    baihu_zone = sector(candidate, facing, facing + 90, radius=aspect_radius)
    bh_height = max_height(baihu_zone)
    bh_roughness = std_height(baihu_zone)

    # 3. 朱雀（前方 90° 扇区）
    zhuque_zone = sector(candidate, facing + 135, facing + 225, radius=aspect_radius)
    zq_slope = mean_slope(zhuque_zone)
    zq_openness = openness_index(zhuque_zone)  # 视野开阔度

    # 4. 玄武（后方 90° 扇区）
    xuanwu_zone = sector(candidate, facing + 225, facing + 315, radius=aspect_radius)
    xw_height = max_height(xuanwu_zone)
    xw_slope = mean_slope(xuanwu_zone)

    # 5. 评分
    # 5.1 青龙分：高而有情
    score['qinglong'] = (
        0.6 * normalize(ql_height, xw_height) +
        0.4 * (100 - normalize(ql_roughness, 5.0))
    )

    # 5.2 白虎分：低而驯（绝不能高于青龙）
    if bh_height > ql_height:
        score['baihu'] = 30  # 凶
    else:
        score['baihu'] = 100 * (1 - bh_height / ql_height)

    # 5.3 朱雀分：开阔
    score['zhuque'] = (
        0.5 * (100 - normalize(zq_slope, 25.0)) +
        0.5 * zq_openness
    )

    # 5.4 玄武分：高而稳
    score['xuanwu'] = (
        0.7 * normalize(xw_height, candidate.elevation + 100) +
        0.3 * (100 - normalize(xw_slope, 30.0))
    )

    return score
```
