## 1. 开源项目结论（可代码级参考）

### 1.1 limi124/shanshui-mingtang-fengshui-gis ⭐ 主参考

**路径：** `third_party/shanshui-mingtang-fengshui-gis/`

| 模块 | 文件 | 算法本质 |
|------|------|----------|
| 综合分 | `backend/app/services/fengshui_score.py` | 围栏级 **标量规则**，非像素四象点位 |
| 地形 | `terrain_analysis.py` | DEM clip → elev / slope / aspect / relief / roughness |
| 水系 | `water_analysis.py` | 最近水体距离 + 是否相交 → 得水分 |

**格局映射（非点选四象）：**

| 传统概念 | 开源指标 | 打分思路（摘要） |
|----------|----------|------------------|
| 靠山（≈玄武） | `back_mountain_score` | 起伏越大越好，坡过陡扣分：`68 + min(relief/4,20) - max(slope-15,0)` |
| 明堂（≈朱雀） | `front_open_score` | 坡度与粗糙度越低越好：`82 - slope*1.1 - roughness*12` |
| 藏风/龙虎 | `enclosure_score` | 有一定起伏、粗糙度不宜过高 |
| 得水 | `water_score` | 见下表（**关键：禁水/近水**） |
| 采光 | `aspect_light_score` | 南/东南/西南优 |
| 稳定 | `terrain_stability_score` | 坡度越低越好 |

**水系距离评分（直接可迁入评分场）：**

| 条件 | 得分 | 含义 |
|------|------|------|
| 与水体 **相交** | 62 | 视觉有水但建设/淹没风险，**不宜当最高穴** |
| ≤ 50 m | 68 | 过近 |
| 50–1000 m | **86** | **最佳近水带** |
| 1000–3000 m | 78 | 尚可 |
| 3000–8000 m | 66 | 偏弱 |
| > 8000 m | ≤58 递减 | 远 |

**对本项目的启示：**

1. **穴不可在水面上**：`intersects` 只能中性偏低，绝不能成为 `argmax` 赢家。  
2. **最佳带是「近水而不贴水」**：约 50–1000 m（与本仓库速查卡「水系最佳 50–1000 m」一致）。  
3. 开源 **没有** 少祖/玄武/青龙/白虎的 **像素级峰值定位**——我们的 `detect_four_beasts` 已更进一步，应 **保留点位算法**，补齐开源的水系硬约束。  
4. 靠山用 **relief + 不宜过陡**，不是「越高越远越好」。

## 2. 传统与文献量化（玄武 / 白虎重点）

来源综合：`00_四兽量化规则.md`、速查卡、AnySearch（葬经/父母山实战文、阴宅选址细则等）。

### 2.1 玄武 = 父母山（近靠），不是远祖

| 层级 | 距穴典型 | 角色 |
|------|----------|------|
| 太祖 / 祖山 | 数 km–10 km+ | 来龙源头 |
| 少祖 | 约 1–5 km | 过渡 |
| **父母山 / 玄武** | **约 50–500 m**（多文强调 **200–500 m 最佳**；阴宅也有 50–100 m 说） | **直接靠山** |
| 穴 | — | 结穴点 |

**玄武吉形（算法可检）：**

- 坐向后方扇区（±35°–45°）  
- 高于穴约 **+30～100 m**（过高逼压、过低脱气）  
- 坡度约 **5°–25°**（「垂头」缓降，忌悬崖 >35°）  
- 圆润完整，忌刀削碎裂  
- **不得落在水体 mask 上**

**反例（当前引擎易犯）：** 穴在河上 → 对岸任意小丘当玄武；或 `weight_elev` 过大选到 1 km+ 远峰。

### 2.2 白虎：低而驯，严禁抬头

| 规则 | 建议阈值 |
|------|----------|
| 相对青龙高度 | **白虎/青龙 ≤ 0.85**；>1.0 为抬头凶 |
| 距离 | 约 **0.5–2 个穴距**，实现上宜 **80–800 m** 为主搜（随 AOI 缩放） |
| 形态 | 缓丘、圆顺；忌尖射、陡尖（坡度过大） |
| 与玄武 | 青龙/玄武 ≤ 0.95（龙不过主山） |

**算法顺序（建议）：**

```text
穴 → 朝向 → 玄武 → 青龙 → 白虎(受青龙高约束) → 朱雀 → 少祖
```

白虎目标函数应 **惩罚过高**，不能与玄武共用「相对高差越大越好」。

### 2.3 朱雀 / 明堂

- 前方开阔：扇区平均坡度小（<8°–12°）  
- 案山 200–1000 m，高度 ≤ 0.5×靠山  
- 朝山 1–5 km，秀而不欺主  
- **水可作朱雀**，但穴本身仍须在陆上

### 2.4 葬经口诀（几何语义）

> 玄武垂头，朱雀翔舞，青龙蜿蜒，白虎驯俯。

| 象 | 几何语义 |
|----|----------|
| 垂头 | 靠山向穴缓降 |
| 翔舞 | 前场开敞、案朝有情 |
| 蜿蜒 | 左侧砂有起伏、分形/曲折 |
| 驯俯 | 右侧砂低伏 |

## 4. 建议落地规格（实现时按此改）

### 4.1 评分场（解决落河 + 橙心）

```text
raw = 0.4*四象扇区 + 0.25*form + 0.2*stability + 0.15*water_kernel
water_kernel:
  on/in water (dist=0 or intersects) → 0 或 nan（硬禁）
  dist < 50 m → 低权
  50–1000 m → 高权（参考 shanshui 86 分带）
  > 3000 m → 衰减
S = gaussian(raw)
穴 = argmax(S on dry land)
热力 PNG = 可视化(S)，peak 标记 = 同一穴
```

### 4.2 玄武选择

```text
主搜: dist ∈ [50, 500] m, 坐向扇区 half=40°
硬: elev ≥ 穴 + 15~30 m; 不在 water_mask
优: Δh ∈ [30, 120], slope ∈ [5, 25]
回退: 放宽至 800–1200 m，仍禁边角与水体
目标: s = w_dist*ideal_dist + w_dh*bell(Δh) - w_steep*steep_penalty
      （不要单调 elev 越大越好）
```

### 4.3 白虎选择

```text
主搜: dist ∈ [80, 800] m, 右侧扇区
硬: elev_baihu_rel ≤ 0.85 * elev_qinglong_rel（相对穴高差）
优: 中低丘、缓坡
目标: 惩罚高差过大 + 奖励理想距离；禁止「越高越好」
```

### 4.4 验收（阆中 3–5 km AOI）

- [ ] 场评最高点不在河面 / 不在 water buffer  
- [ ] 橙心与场评最高点像素一致（偏差 < 1–2 像元）  
- [ ] 玄武距离多在 50–500 m，形态为近靠丘  
- [ ] 白虎不高于青龙；距离显著短于 1.5–2 km 远峰乱抓
